#### Test the Flash Sale Ticket API

After successfully deploying **Amazon API Gateway**, the next step is to verify that the system can receive ticket booking requests. The API is tested using **Postman** to ensure that it accepts user requests and successfully forwards them to **Amazon SQS**.

The purpose of this API is to receive Flash Sale ticket booking requests from users and place them into **Amazon SQS** for asynchronous processing by **AWS Lambda**. This architecture provides a fast response time because users do not need to wait for the system to write data directly to the database.

### Request Configuration

Configure the request in Postman using the following settings:

- **HTTP Method:** `POST`
- **Request URL:**

```text
https://7c4kyholj0.execute-api.ap-southeast-1.amazonaws.com/prod/buy
```

- **Headers**

```text
Content-Type: application/json
```

### Request Body

Select **Body** → **raw** → **JSON**, then enter the following content:

```json
{
    "ticketId": 1,
    "userId": 8888
}
```

Parameter descriptions:

- **ticketId:** The unique identifier of the ticket to be booked (for example, `1` represents a VIP ticket).
- **userId:** The unique identifier of the user making the booking request.

### Verify the Response

After clicking **Send**, if the system is working correctly, Postman should return the following status code:

```text
Status: 200 OK
```

The response body will be similar to the following:

```json
{
  "SendMessageResponse": {
    "ResponseMetadata": {
      "RequestId": "2bebfec7-a03a-5b93-8922-9c1e650030d4"
    },
    "SendMessageResult": {
      "MD5OfMessageBody": "799e1eb079ffb4d24bccff0b23f52ed75",
      "MessageId": "11dd0785-a895-48b9-a180-d2e0fd3adfc3"
    }
  }
}
```

![postman-response](/images/5-Workshop/5.3-S3-vpc/17.jpg)

### Response Explanation

The response fields have the following meanings:

- **SendMessageResponse:** Confirms that Amazon SQS has successfully received the request.
- **RequestId:** A unique identifier for the request generated by AWS API Gateway.
- **MessageId:** The unique identifier assigned to the message in the Amazon SQS queue. This confirms that the ticket booking request has been successfully queued for processing by AWS Lambda.
- **MD5OfMessageBody:** The MD5 hash of the message body, used to verify data integrity during transmission.

When the API returns **HTTP Status 200 OK** along with a valid **MessageId**, it confirms that **Amazon API Gateway** has successfully forwarded the booking request to **Amazon SQS**. AWS Lambda will then automatically retrieve the message from the queue and continue processing the ticket booking request according to the system's **Serverless Event-Driven Architecture**.
