### Week 5 Objectives

* Learn the fundamentals of AWS Identity and Access Management (IAM) and access control.
* Practice creating IAM Users, IAM Groups, IAM Roles, and performing Assume Role operations.
* Apply IAM Conditions to enhance security and control access to AWS resources.

### Week 5 Achievements

* Gained a solid understanding of AWS Identity and Access Management (IAM), including Users, Groups, Roles, Policies, and the AWS request authorization process.

* Learned how the Assume Role mechanism works through AWS Security Token Service (STS) and successfully performed role switching using the Switch Role feature.

* Successfully created IAM Groups and four IAM Users with different permission levels based on the required access policies.

* Created an IAM Role with **AdministratorAccess** permissions and configured the appropriate Trust Relationship to allow authorized users to assume the role.

* Successfully applied IAM Conditions to restrict role switching based on source IP addresses and access time.

* Verified and tested the permissions assigned to each IAM User to ensure that access control policies worked as expected.

* Cleaned up all IAM resources, including IAM Users, Groups, and Roles, after completing the hands-on lab.

* Gained practical experience with the **Principle of Least Privilege** and learned how to strengthen AWS security using **context-based access control** with IAM Conditions.
